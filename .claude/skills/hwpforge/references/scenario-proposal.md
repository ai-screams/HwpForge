# 정부 제안서 (Government Proposal) 시나리오

> **Note**: This scenario references the `government` preset which is **planned but not yet available**. Use `default` preset until it is implemented.

## 표준 구조

한국 정부 발주 RFP에 대응하는 제안서의 표준 목차 구성입니다.

```
1. 표지 (Cover Page)
2. 목차 (Table of Contents)
3. 사업 이해 (Understanding of the Project)
   3.1 사업 배경 및 목적
   3.2 현황 분석
   3.3 문제점 및 개선 방향
4. 수행 방안 (Implementation Plan)
   4.1 추진 전략
   4.2 세부 수행 계획
   4.3 주요 산출물
5. 수행 조직 (Project Organization)
   5.1 프로젝트 조직도
   5.2 투입 인력 현황
   5.3 핵심 인력 이력
6. 관리 방안 (Management Plan)
   6.1 일정 관리
   6.2 품질 관리
   6.3 위험 관리
7. 참고 자료 (References)
```

## 마크다운 규칙

- `#` (H1): 과업명 / 문서 제목 (표지에만 사용)
- `##` (H2): 장 번호 (`## 1. 사업 이해`)
- `###` (H3): 절 번호 (`### 1.1 사업 배경`)
- `####` (H4): 항 번호 (`#### 1.1.1 현황`)
- GFM 표: 인력 현황, 일정표, 비교표에 사용
- `---`: 장(chapter) 구분 시 페이지 분리 신호

## 프리셋

```bash
hwpforge convert proposal.md -o proposal.hwpx --preset government
```

`government` 프리셋 설정:

- 제목 글꼴: 고딕체 16pt Bold
- 줄간격: 160%
- 여백: 위/아래 20mm, 좌/우 25mm
- 용지: A4 세로

# AI 기반 문서 관리 시스템 구축 사업 제안서

제안사: (주)에이아이스크림
제출일: 2026년 3월 9일
수신: ○○부 정보화담당관

---

## 1. 사업 이해

### 1.1 사업 배경 및 목적

현재 공공기관의 문서 관리는 수기 분류 및 수동 검색에 의존하고 있어
처리 시간 증가, 분류 오류, 중복 문서 발생 등의 문제가 지속되고 있습니다.

본 사업은 AI 기반 자동 분류·검색 시스템을 구축하여 문서 처리 효율을
70% 향상시키고, 정확도를 95% 이상으로 개선하는 것을 목적으로 합니다.

### 1.2 현황 분석

| 구분             | 현황         | 목표        |
| ---------------- | ------------ | ----------- |
| 문서 분류 시간   | 평균 15분/건 | 평균 2분/건 |
| 분류 정확도      | 82%          | 95% 이상    |
| 중복 문서 발생률 | 12%          | 3% 이하     |
| 연간 처리 건수   | 약 50,000건  | 약 70,000건 |

---

## 2. 수행 방안

### 2.1 추진 전략

본 사업은 3단계 접근법으로 추진합니다.

1. **1단계 (1~3개월)**: 현행 시스템 분석 및 AI 모델 설계
2. **2단계 (4~8개월)**: 시스템 개발 및 파일럿 운영
3. **3단계 (9~12개월)**: 전면 전환 및 안정화

### 2.2 세부 수행 계획

#### 2.2.1 AI 분류 엔진 개발

- 자연어 처리(NLP) 기반 문서 자동 분류 모델 구현
- 공공기관 문서 유형 200종 학습 데이터셋 구축
- 정확도 목표: 95% 이상 (F1-score 기준)

#### 2.2.2 검색 시스템 구축

- 시맨틱 검색 엔진 도입 (벡터 유사도 기반)
- 검색 응답 시간: 1초 이내 목표
