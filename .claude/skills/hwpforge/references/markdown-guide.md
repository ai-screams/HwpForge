# 한국어 마크다운 작성 규칙

HwpForge가 올바르게 변환하려면 GFM(GitHub Flavored Markdown) 표준을 따르되, 한국어 문서 특성에 맞는 규칙을 지켜야 합니다.

## YAML 프론트매터

모든 문서의 맨 앞에 YAML 블록으로 메타데이터를 지정합니다.

```yaml
---
title: "문서 제목"
author: "작성자 이름"
date: "2026-03-09"
preset: government
---
```

지원 필드:

| 필드           | 설명          | 예시                                          |
| -------------- | ------------- | --------------------------------------------- |
| `title`        | 문서 제목     | `"AI 기반 시스템 구축 제안서"`                |
| `author`       | 작성자        | `"홍길동"`                                    |
| `date`         | 작성일        | `"2026-03-09"`                                |
| `preset`       | 스타일 프리셋 | `government`, `report`, `official`, `default` |
| `organization` | 소속 기관     | `"(주)에이아이스크림"`                        |

프론트매터가 없으면 `default` 프리셋이 적용됩니다.

## 제목 계층 구조

```markdown
# 문서 제목 (H1 — 표지 또는 최상위 제목)

## 1. 장 제목 (H2 — 주요 섹션)

### 1.1 절 제목 (H3 — 하위 섹션)

#### 1.1.1 항 제목 (H4 — 세부 항목)
```

주의사항:

- 번호는 마크다운에서 수동 작성 (`## 1.`, `### 1.1`)
- 계층을 건너뛰지 말 것 (H2 다음 바로 H4 금지)

## GFM 표 문법

```markdown
| 열1    | 열2    | 열3    |
| ------ | ------ | ------ |
| 데이터 | 데이터 | 데이터 |
| 데이터 | 데이터 | 데이터 |
```

정렬 지정:

```markdown
| 왼쪽 정렬 | 가운데 정렬 | 오른쪽 정렬 |
| :-------- | :---------: | ----------: |
| 텍스트    |   텍스트    |        숫자 |
```

한국어 표 작성 팁:

- 헤더 셀에 **볼드** 사용 가능: `| **항목** |`
- 합계 행에는 볼드 강조: `| **합계** | **100%** |`
- 숫자는 오른쪽 정렬 권장

## 이미지 삽입

```markdown
![대체 텍스트](./images/figure1.png)

**[그림 1] 시스템 아키텍처**
```

경로 규칙:

- 절대 경로도 지원: `/Users/user/project/images/fig1.png`

- 지원 형식: PNG, JPG, JPEG, GIF, BMP, SVG

이미지 디렉토리를 마크다운 파일과 함께 유지하세요:

```
project/
├── document.md
└── images/
    ├── figure1.png
    └── figure2.png
```

## 페이지 구분 (수평선)

`---` (하이픈 3개 이상)은 장(chapter) 사이의 페이지 구분으로 변환됩니다.

```markdown
## 제1장 서론

본문 내용...

---

## 제2장 현황 분석

본문 내용...
```

주의: `---`는 반드시 빈 줄로 둘러싸야 합니다.

## 강조 문법

```markdown
**볼드 텍스트** — 중요 키워드, 표 헤더
_이탤릭 텍스트_ — 영문명, 외래어 (한국어에는 거의 미사용)
~~취소선~~ — 삭제된 내용
`코드` — 시스템 명칭, 파라미터, 명령어
```

## 목록 문법

순서 없는 목록:

```markdown
- 항목 1
- 항목 2

  - 하위 항목 (공백 2칸 들여쓰기)
  - 하위 항목
```

순서 있는 목록:

```markdown
1. 첫 번째
2. 두 번째
3. 세 번째
```

공문서 번호 체계 (수동 작성):

```markdown
1. 첫째 수준

   가. 둘째 수준

   1. 셋째 수준

      가) 넷째 수준
```

## 각주 문법

```markdown
본문 내용입니다.[^1] 추가 내용.[^ref]

[^1]: 각주 내용 (문서 하단에 자동 배치됨)

[^ref]: 참고문헌 각주
```

각주는 HWPX 변환 시 한글 각주 형식으로 자동 변환됩니다.

## 코드 블록

````markdown
```python
# 코드 블록 예시
def hello():
    print("안녕하세요")
```
````

정부 문서에서는 주로 시스템 명령어, SQL, 설정값 표시에 사용합니다.

## 특수문자 처리

마크다운에서 이스케이프가 필요한 문자:

| 문자 | 이스케이프 | 용도                                |
| ---- | ---------- | ----------------------------------- |
| `*`  | `\*`       | 별표를 볼드/이탤릭 아닌 문자로 사용 |
| `_`  | `\_`       | 밑줄 문자 그대로 사용               |
| `[`  | `\[`       | 대괄호를 링크 아닌 문자로 사용      |
| `#`  | `\#`       | 샵을 제목 아닌 문자로 사용          |

한국어 특수문자는 이스케이프 없이 직접 입력합니다: ①②③, ㄱㄴㄷ, ◆◇●

## 인코딩 및 개행

- 파일 인코딩: UTF-8 (BOM 없음)
- 줄바꿈: LF (Unix 방식) 권장, CRLF도 지원
- 문단 구분: 빈 줄 1개

긴 문장은 가독성을 위해 80자 내외에서 줄바꿈하되, HWPX 변환 시 같은 문단으로 처리됩니다.

```markdown
이 문장은 마크다운에서 두 줄로 작성되었지만
HWPX에서는 하나의 문단으로 연결됩니다.

빈 줄이 있어야 새 문단으로 구분됩니다.
```
