# 공문서 (Official Document) 시나리오

> **Note**: This scenario references the `official` preset which is **planned but not yet available**. Use `default` preset until it is implemented.

## 표준 구조

행정업무운영규정(대통령령)에 따른 공문서 구성입니다.

```
두문 (Header)
  - 기관명 (발신 기관)
  - 수신자 (수신 기관 및 참조)

본문 (Body)
  - 제목
  - 내용

결문 (Footer)
  - 발신 명의 (기관장명)
  - 기안자 / 검토자 / 결재자 직위·서명
  - 시행 번호 및 시행일
  - 접수 번호 및 접수일
  - 우편번호·주소 / 홈페이지 / 전화·팩스
  - 공개 구분 (공개, 부분공개, 비공개)
```

## 마크다운 규칙

번호 체계는 행정업무운영규정 제3조 별표1 기준을 따릅니다.

```
1.     (첫째 수준 — 항목)
  가.  (둘째 수준)
    1) (셋째 수준)
      가) (넷째 수준)
```

마크다운 표현:

```markdown
1. 첫째 수준 항목

   가. 둘째 수준 항목

   1. 셋째 수준 항목

      가) 넷째 수준 항목
```

주의: 한글 워드프로세서의 자동 번호 매기기와 달리 마크다운은 수동으로 번호를 작성합니다.

## 프리셋

```bash
hwpforge convert official.md -o official.hwpx --preset official
```

`official` 프리셋 설정:

- 제목 글꼴: 고딕체 12pt Bold
- 줄간격: 180%
- 여백: 위 30mm, 아래 15mm, 좌/우 20mm
- 쪽번호: 없음 (공문서 표준)
- 용지: A4 세로

## 공문서 작성 핵심 규칙

### 날짜 표기

- 공문서 표준: `2026. 3. 9.` (마침표 뒤 공백, 연도/월/일 모두 마침표)
- 숫자만 사용, 한자 혼용 금지

### 단위 기호

- 숫자 + 단위: `3건`, `5부`, `1식`
- 금액: `금 1,000,000원` (또는 `금 일백만원`)

### "끝" 표기

- 본문 마지막 항목 끝에 반드시 `끝.` 을 붙임
- 붙임 자료가 있으면 붙임 목록 마지막에 `끝.`

### 경어 사용

- 지시문: `~하시기 바랍니다`, `~하여 주시기 바랍니다`
- 통보문: `~알려드립니다`, `~통보합니다`
- 요청문: `~협조하여 주시기 바랍니다`
